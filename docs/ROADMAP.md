優先度 1〜7 は実装済み。ここでは**まだ実装していない** 8〜11 の設計だけを残す。

- PC / スマホの両方を壊さない（サイドバー ⇄ 下部ナビの切替は `lg`）

- 結果画面の二重加算防止（`useSessionResult` の3段構え）を壊さない

## 8. 提携店舗のQRチェックイン風デモ

**狙い**：「提携店舗で使う」という体験がイメージできること。QRは本物でなくてよい。

- 新規ルート `/checkin`（セッション作成の前段）
- 利用形態で「提携店舗 / 直営店舗」を選んだときだけ、作成画面に「店舗でチェックインする」ボタンを出す
- 画面構成：カメラ風の枠 → 「QRを読み取りました」演出（1.5秒のローディング）→ 店舗名・座席・特典を表示 → セッション作成へ戻る
- 店舗データは `src/data/stores.ts` にモックで持つ（店舗名・住所・レート倍率・特典テキスト）
- チェックイン結果は `SessionConfig.storeId?`（**任意フィールド**）として保存し、集中画面・結果画面・履歴に店舗名を出す

**想定変更ファイル**：`src/pages/CheckinPage.tsx`（新規）／`src/data/stores.ts`（新規）／`SessionCreatePage.tsx`／`types/index.ts`／`App.tsx`

**リスク**：カメラ API は使わない（許可ダイアログでデモが止まる）。**擬似的な読み取り演出に留める**こと。

## 9. ロッカー番号・スタッフ確認画面

**狙い**：直営店舗の「物理的にスマホを封印する」オペレーションを見せる。Web版の制約（他アプリを止められない）に対する答えになるので、面接での説得力は高い。

- チェックイン後に「ロッカー番号 A-07」を払い出す画面（`/locker`）
- セッション中の集中画面に「ロッカー A-07 に封印中」を常時表示
- スタッフ確認画面 `/staff`（デモ用の別モード）：本日の封印中セッション一覧と「返却する」ボタン
- ロッカー番号は `SessionConfig.lockerId?` として保存（任意フィールド）

**想定変更ファイル**：`src/pages/LockerPage.tsx`／`src/pages/StaffPage.tsx`（新規）／`FocusSessionPage.tsx`／`types/index.ts`／`App.tsx`

**リスク**：スタッフ画面をサイドバーに常設すると学生向けアプリとして不自然。**設定画面から入る「店舗スタッフ用（デモ）」の隠し導線**にする。

## 10. DP交換画面

**狙い**：貯めた DP の出口（収益モデル）を見せる。

- 新規ルート `/exchange`
- 交換アイテムをカード一覧で表示（例：提携カフェ ドリンク1杯 300DP／自習席1時間無料 500DP／文房具 200DP）
- 交換すると `userStats.currentDP` を減算し、`dopalock:exchanges` に履歴を保存
- **DP不足のときは押せない**（disabled + 不足分の表示）

**想定変更ファイル**：`src/pages/ExchangePage.tsx`（新規）／`src/data/rewards.ts`（新規）／`utils/storage.ts`（`exchanges` キー追加）／`Sidebar.tsx`／`App.tsx`

**リスク**：DP 減算はレート計算と独立させる。**`applySessionToStats` には触らない**こと（セッションの二重加算防止ロジックと混ぜない）。

## 11. 協力・バトルモードの待機ルーム

**狙い**：友達と競う体験。ただし**実装コストが最も高く、バックエンドが無い以上「演出」にしかならない**ため優先度は最下位。

- セッション作成で協力／バトルを選んだとき、開始前に待機ルーム `/session/lobby` を挟む
- 参加者は `src/data/friends.ts` のモック。開始ボタンで「マッチしました」演出（2秒）→ 集中画面へ
- バトル中は集中画面に相手の進捗バーを出す（**タイマーと同じ経過秒から算出した擬似進捗**）
- 結果画面に勝敗（自分が成功 かつ 相手より長ければ勝ち、など単純なルール）を出す

**想定変更ファイル**：`src/pages/LobbyPage.tsx`（新規）／`src/data/friends.ts`（新規）／`FocusSessionPage.tsx`／`Success/JailbreakResultPage.tsx`／`App.tsx`

**リスク**：**セッション中のルートガード（`AppLayout` の `SESSION_ROUTES`）に `/session/lobby` を足さないこと**。ロビーは開始前の画面なので、runtime を作る前に通す。ここを間違えると無限リダイレクトになる。
