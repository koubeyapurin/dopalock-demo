# DopaLock デモアプリ仕様書

本書は DopaLock デモアプリの実装基準です。実装で迷ったらこの仕様を優先します。

---

## 1. 目的

LEVEL UP STAGE 2026 の面接で、DopaLock のサービス体験を説明するためのデモ Web アプリ。
審査員に、以下を直感的に伝える。

- スマホを封印して集中するサービスであること
- 集中成功でレート・DP が貯まること
- 途中でスマホを見ると「脱獄」になること
- 5 分チケットで短い休憩ができること
- 個人・協力・バトルなどのモードがあること
- 実証実験で使える形に近いこと

---

## 2. 想定ユーザー

大学生。
テスト勉強・レポート・資格勉強・就活準備をしたいが、スマホ・SNS・YouTube・ショート動画に集中を奪われてしまう人。

---

## 3. デモで見せたい体験（コアフロー）

1. ホーム画面で現在レート・DP・集中時間・成功回数が見える
2. セッション作成画面で、利用形態・モード・時間を選べる
3. セッションを開始すると、スマホロック中の集中タイマーが表示される
4. 途中で 5 分チケットを使うと、5 分休憩タイマーになる
5. 休憩後、集中に戻れる
6. 完了すると成功結果画面に移動し、レート・DP が増える
7. 脱獄ボタンを押すと脱獄結果画面に移動し、予定レートの 80% を失う
8. ダッシュボードで実績・履歴・ランキングが見える

## 5. 計算ルール

### 5.1 基本値
- 基本レート：`2.0`
- 基本 DP：`6`
- 標準時間：`60 分`

### 5.2 利用形態倍率
| 利用形態 | 倍率 |
|---|---|
| どこでも DopaLock | 1.0 |
| 提携店舗 DopaLock | 1.2 |
| 直営店舗 DopaLock | 1.5 |

### 5.3 モード倍率
| モード | 倍率 |
|---|---|
| 個人モード | 1.0 |
| 協力モード | 1.1 |
| バトルモード | 1.3 |

### 5.4 成功時
```
予定獲得レート = 2.0 × (時間 / 60) × 利用形態倍率 × モード倍率
予定獲得DP     = 6   × (時間 / 60) × 利用形態倍率 × モード倍率
```

### 5.5 脱獄時
```
失うレート = 予定獲得レート × 0.8
獲得DP     = 0
```

### 5.6 5 分チケット
- 初期残数は 2 枚
- 使用すると 1 枚減る
- セッション中のみ使える
- 5 分休憩後は集中に戻れる
- チケットが 0 枚の場合は使用不可

### 5.7 計算例（検算用）
表示は小数第 1 位を目安に丸める。

| 利用形態 | モード | 時間 | 予定レート | 予定DP |
|---|---|---|---|---|
| どこでも | 個人 | 60 分 | 2.0 | 6 |
| どこでも | 個人 | 120 分 | 4.0 | 12 |
| 提携店舗 | 協力 | 60 分 | 2.6 | 7.9 |
| 直営店舗 | バトル | 90 分 | 5.9 | 17.6 |

計算式：`2.0 × 90/60 × 1.5 × 1.3 = 5.85`、`6 × 90/60 × 1.5 × 1.3 = 17.55`

## 6. データ構造

### 6.1 UserStats
| フィールド | 型 | 説明 |
|---|---|---|
| name | string | ユーザー名 |
| grade | string | 学年（例：大学2年生） |
| currentRate | number | 現在レート |
| currentDP | number | 現在 DP |
| tickets | number | 5 分チケット残数 |
| totalFocusMinutes | number | 累計集中時間（分） |
| successCount | number | 成功回数 |
| jailbreakCount | number | 脱獄回数 |
| streakDays | number | 連続成功日数 |

### 6.2 SessionConfig
| フィールド | 型 | 説明 |
|---|---|---|
| usageType | 'anywhere' \| 'partner' \| 'direct' | 利用形態 |
| mode | 'personal' \| 'coop' \| 'battle' | モード |
| durationMinutes | 30 \| 60 \| 90 \| 120 | セッション時間 |
| plannedRate | number | 予定獲得レート |
| plannedDP | number | 予定獲得 DP |

### 6.3 SessionRecord
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意 ID |
| date | string | 実施日時（ISO 文字列） |
| usageType | 'anywhere' \| 'partner' \| 'direct' | 利用形態 |
| mode | 'personal' \| 'coop' \| 'battle' | モード |
| durationMinutes | number | 予定時間（分） |
| result | 'success' \| 'jailbreak' | 結果 |
| rateChange | number | レート増減 |
| dpChange | number | DP 増減 |
| usedTicket | boolean | 5 分チケット使用有無 |
| actualMinutes | number | 実際の集中時間（分） |

## 7. 実装方針

- バックエンドなし
- 認証なし
- localStorage に `UserStats` と `SessionRecord[]` を保存する
- 初期データは `src/data/mockData.ts` に置く
- デモ用なのでタイマーは実時間より短縮できる設定を用意してもよい（デモ加速モード）
- ただし**画面上は 30 / 60 / 90 / 120 分として表示**する
- 全ボタンが本番機能を持つ必要はないが、第 3 章のコアフローは必ず動かす
