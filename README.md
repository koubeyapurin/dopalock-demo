# DopaLock デモアプリ

スマホを触らない時間を「レート・DP・友達バトル・店舗利用特典」に変える、大学生向けのスマホ封印型自習サービス **DopaLock** の体験デモです。
LEVEL UP STAGE 2026 の面接審査で、サービス体験を伝えるためのプロトタイプです。

## 技術スタック
React + TypeScript / Vite / Tailwind CSS / react-router-dom / lucide-react / recharts / localStorage

## セットアップ

```bash
npm install    # 依存インストール
npm run dev     # 開発サーバー起動（表示された http://localhost:5173 を開く）
npm run build   # 本番ビルド
```

バックエンド・認証はありません。データはすべて **モックデータ + localStorage** で管理します。

## 面接で見せるおすすめ操作手順

> 事前に **設定 → デモデータをリセット** で初期状態にしておくと、毎回きれいな状態から始められます。
> タイマーは「タイマー短縮モード」ON（初期値）で、表示1分がおよそ実3秒で進みます。

### 1. ホーム（トップ）
- 「スマホを触らない時間を、集中実績に変える」というコンセプトを説明
- 現在レート・DP・今日の集中時間・成功回数のサマリーを見せる
- **「セッション開始」** をクリック

### 2. セッション作成
- **利用形態**（どこでも／提携店舗／直営店舗）を選ぶ
- **モード**（個人／協力／バトル）を選ぶ
- **時間**（30／60／90／120分）を選ぶ
- 右カードで **予定獲得レート・DP がリアルタイムに変わる** ことを見せる
- **「セッション開始」** をクリック

### 3. 集中セッション中
- スマホロック中の集中タイマー（円形）を見せる
- 予定獲得レート・DP、集中ステータス、チケット残数を説明

#### 3-a. 5分チケットで休憩（サポート機能）
- **「5分チケットを使う」** → 休憩画面（5:00 のタイマー）
- 「集中に戻る」で集中へ復帰（経過とチケット残数が引き継がれる）

### 4. 結果を見せる（2パターン）
- **成功**：集中セッション中で **「完了する」** → セッション成功画面（レート・DP が加算）
- **脱獄**：**「脱獄する」** → 脱獄画面（**予定レートの80%を失う**／DPは0）

> ※ 成功と脱獄を両方見せると、サービスのインセンティブ設計が伝わります。

### 5. ダッシュボードに反映
- 結果画面の **「ダッシュボードを見る」**、またはサイドバーから移動
- 直近7日間の集中時間グラフ・成功率ドーナツ・履歴テーブル・ランキングに **さっきのセッションが反映** されていることを見せる

### 6. その他の画面（補足）
- **ルール説明**：レート/DP/チケット、利用形態、モード、脱獄ペナルティ、初期実証（調布周辺）
- **ランキング**：自分の順位を強調
- **履歴**：全セッションの一覧
- **チケット**：残数・説明（デモ用に +1 可能）

## デモ運用のヒント（設定画面）
- **タイマー短縮モード**：ON=デモ用に高速、OFF=ほぼ実時間
- **デモデータをリセット**：実績・履歴・進行中セッションを初期化して最初からやり直す

## 主要な計算ルール
- 予定獲得レート = `2.0 × 時間/60 × 利用形態倍率 × モード倍率`
- 予定獲得DP = `6 × 時間/60 × 利用形態倍率 × モード倍率`
- 利用形態倍率：どこでも 1.0／提携 1.2／直営 1.5
- モード倍率：個人 1.0／協力 1.1／バトル 1.3
- **脱獄時：予定レートの80%を失い、DPは獲得なし**

詳細な仕様は [docs/SPEC.md](docs/SPEC.md) を参照してください。
